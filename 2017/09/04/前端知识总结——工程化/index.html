前端知识总结——工程化

发表于 2017-09-04

模块化开发的多模块文件与线上要求请求数少是矛盾的，因此工程化的解决方案是：两套代码，一套线下的开发代码，一套线上的部署代码。

工程化

模块化需要将海量的代码拆分成不同的模块，而HTTP请求数又是十分影响性能的，这两者的矛盾就是开发环境和线上环境的矛盾。工程化就是为了解决这个矛盾的。

  • 开发时可以用模块化开发，可以使用Sass、TypeScript来提高开发效率；
  • 准备部署上线时，可以将Sass编译成CSS、TypeScript编译成JavaScript，并进行压缩代码、合并文件。

自动化的处理这些过程就是工程化。当然了，既然是工程化，肯定不是手动完成所有了，需要借助一些工程化的工具。

构建

由于开发环境和线上环境的需求是不一样的：

  • 开发环境（源代码）
    • 按模块划分多文件
    • 代码可读性好，需要注释
    • 编写最简单的代码
  • 线上环境（线上代码）
    • 文件越少越好
    • 代码越短越好
    • 兼容不同浏览器

从源代码转为线上代码，需要经过编译、打包、压缩、优化等操作，而自动化地完成这些操作就叫做构建。

常见构建工具对比

GRUNT

  • 优点
    • 基于Node
    • 上手容易
    • 插件齐全
    • 社区活跃
  • 缺点
    • 每次操作都是文件读写，速度较慢
    • 配置相对繁琐

Gulp

可以说Gulp是对GRUNt的缺点进行改进的产物。

  • 优点：
    • 配置相对简单
    • 操作基于文件流，速度较快

webpack

webpack一开始的定位只是一个打包工具，针对打包进行了很多优化:

  • 任何文件都可以当模块
  • 打包体积更小
  • 打包更快
  • 默认支持AMD、CommonJS、ES6模块化标准

刚开始，大家都是在Gulp的基础上使用gulp-webpack插件进行打包。随着webpack的流行，各种围绕webpack的插件开始丰富，不再需要Gulp，已经自成体系，即webpack构建工具。

webpack入门

安装

目前前端使用的构建工具都是用 Node.js 写的，这样有助于我们前端才能更好地去了解和使用这些工具。因此，可以通过npm下载安装。

npm install webpack --save-dev

  • --save
    • 将产品生产环境需要的npm包存入到package.json的dependencies
  • --save-dev
    • 将产品开发环境需要的npm包存入到package.json的devDependencies

检测是否安装成功：

webpack -v

webpack简单使用

webpack作为一个命令行工具，可以通过如下命令简单使用：

webpack ./src/index.js ./dist/bundle.js

webpack loader

webpack本身就是一个打包工具，可以根据分析模块间的依赖来打包JavaScript文件，那么如何才能打包CSS、图片等资源呢？

通过使用 css-loader 这个npm包，可以将css文件转为webpack模块。

webpack还有其他loader插件，如：

  • babel-loader（ES6 To js）
  • scss-loader（scss To css）

总的来说，webpack loader 就是针对特定类型的单文件进行相应处理。

通过命令行来安装 css-loader ：

npm install css-loader --save-dev

webpack html 模板

通过前面的构建操作，已经将 JavaScript、CSS 打包并压缩到 ./dist 目录下了，但是项目的 index.html 还是在根目录下，而且页面引用打包后的 js 的路径也是我们手动加上去的。

实际项目中我们希望把 html 文件也加入构建中，并且自动将打包后的 js 加到 html 中，甚至可以使用不同的模板自动生成 html 文件，在 Webpack 里面需要 html-webpack-plugin 插件来实现。

html-webpack-plugin 是 Webpack 的插件，可以用于自动生成 html 文件或者复制 html 文件并且将打包后的 js 加入到 html 中。

webpack本地调试自动刷新

本地开发流程：

  • 保存代码
  • 跑构建
  • 刷新浏览器

这样的流程略显繁琐，而通过使用 webpack-dev-server 可以自动化的完成这个流程。

  • 命令行输入 npm i webpack-dev-server -g 全局安装（也可本地）
    • 全局安装的话，需要确保全局也安装了webpack，因为其依赖webpack
  • 命令行输入 webpack-dev-server --open 开启服务

webpack 使用总结

  • 使用 CommonJS 进行对 src/js 目录下的 js 代码进行模块化，所有模块都不产生全局变量，只通过 require 声明依赖，以及通过 module.exports 暴露模块接口。

  • 根目录增加 webpack.config.js 配置文件，使用 Webpack 对 js 进行打包, 入口文件为 src/js/index.js, 打包输出到 dist/bundle.js。

  • 使用 css-loader 和 style-loader, 将 src/css/style.css 也加入打包。

  • 使用 html-webpack-plugin 将 src/index.html 作为模板，删掉index.html 里面所有的 <script> 和 <link> 标签，最终在 dist/ 目录自动生成引用打包后文件的 index.html 。

  • 使用 copy-webpack-plugin 将 src/images 下的所有图片复制到 dist/images 目录。

  • 使用 webpack.optimize.UglifyJsPlugin 对代码进行压缩。

  • 使用 extract-text-webpack-plugin 将 CSS 文件分离出来，构建后目录单独有一个 style.css 文件。

  • 使用 clean-webpack-plugin， 每次构建之前删掉 dist 目录，避免上一次构建的影响。

  • 使用 webpack-dev-server 可以开启本地服务器，保存代码后页面自动刷新。
