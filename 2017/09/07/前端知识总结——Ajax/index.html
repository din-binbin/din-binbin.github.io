前端知识总结——Ajax

发表于 2017-09-07

通过在后台与服务器进行少量数据交换，AJAX 可以使网页实现异步更新。这意味着可以在不重新加载整个网页的情况下，对网页的某部分进行更新。

Ajax起源

有一种需求，只需要刷新页面的一小部分。

  • iframe 性能和安全性不够好

  • ajax

本地服务器

服务器和PC的主机类似，PC也可以做服务器。

Nginx、Apache、Tomcat

Nodejs（npm : http-server）

XHR

Ajax过程如下：

  • 新建连接 new XMLHttpRequest
  • 打开链接 xhr.open()
  • 发送数据 xhr.send()
  • 接收数据

这是一个异步的过程，当其发送数据的时候JavaScript可以不必等到数据接收完成再运行其他代码，而是通过监听接收状态。如果是同步的话，则必须等到其接收完数据再进行下一步。

通过onreadystatechange事件，来监听xhr对象的readyState状态属性。当readyState===2的时候，表示发送数据；当readyState===3的时候，表示开始接收数据；当readyState===4的时候表示接收完成。

JSON

JSON (JavaScript Object Notation) is a lightweight data-interchange format.

JSON 是一种轻量级的数据交换格式。JSON 是一种格式，而不是对象，所以别再说“这个JSON对象有问题”这种话。

这才是 JSON ，JSON 就是文本，和什么语言没有关系，JS 可以解析，自然 JAVA、Python 都可以解析。

参数传递

get和post

  • HTTP 协议并没有限定 GET/POST 请求参数的长度，长度限制一般都是由浏览器和服务器来决定，所以各浏览器和服务器都不尽相同。GET传参最大长度的理解误区

  • GET 参数更容易被查看、截断、替换等，相比 POST 来说不安全，当然 POST 也安全不到哪里去

构造 GET 参数的时候，总是会通过 encodeURIComponent 编码的，总是会；POST 则不用

  • GET 参数只能加在 URL 的末尾，POST 编码只要和服务器约定好就行了，至少有四种常见的编码。可参考：四种常见的 POST 提交数据方式

同源策略

不同域的客户端脚本再没有明确授权的情况下，不能读写对方的资源。

  • 同域 —— 同协议、同域名、同端口
  • 脚本 —— JavaScript
  • 资源 —— Ajax、cookie

同源策略的初衷就是安全，这个无需多言。但是同源限制也意味着不能共享，这是有悖于互联网的初衷的，所以折中了一下，script image style等等资源都是可以跨域共享的。

如果二级域名相同，可以通过 JS 来提升域名。

cookie不能跨域读取。关于cookie可参考：HTTP cookies 详解

CORS

CORS，Cross Origin Resource Sharing，跨域资源共享。

ifram 结合 window.name

通过<iframe>标签，可在页面内嵌入其他域名的网页。

window.name特点：当在一个页面设置了window.name后，在页面切换后，即使不同域，也会保留window.name的设置。可以借用这一点进行跨域。

举例：现有域名端口为3001下的网页a.html，想获取域名端口为3002下的网页b.html里的数据。

实现方法：

  • 在a.html中嵌入一个iframe，地址为b.html的端口为3002的地址
  • 之后将需要的数据赋值给window.name，再将iframe重定向到端口为3001的c.html
  • 此时通过c.html中的window.name已经可以获取到b.html的数据了，而且，a.html和c.html是同域的，于是a.html也能获取到b.html的数据了
