前端知识总结——浏览器渲染

发表于 2017-08-25

浏览器是前端性能优化不可不说的，重要性杠杠的。

浏览器和渲染引擎

我们所编写的HTML、CSS、JavaScript代码都是在浏览器这样的宿主环境下运行的，因此很有必要了解一下浏览器。

浏览器包括哪些组成部分？

其中，又把HTML解释器、CSS解释器、布局和JS引擎组成的部分称为渲染引擎。

渲染引擎有哪些？

  • Blink （Google Chrome、Opera）
  • Trident （Edge/IE）
  • Webkit （Safari）
  • Gecko （Firefox）

Blink是从Webkit引用过去的，非常相似。

从 HTML 到 DOM 发生了什么

  • 对HTML文档进行词法解析，解析成token（最小单元）
  • 再对token进行扫描，依据语法进行语法解析，最后形成DOM

渲染过程

渲染特点：边加载边解析边渲染。

在HTML解析为DOM的过程中，遇到CSS则会进行CSS解析，经过解析成为样式规则，与DOM树（DOM Tree）一起构成渲染树（Render Tree）。

但是，在HTML和CSS解析为渲染树的过程中，如果遇到JavaScript代码，则会阻塞解析。因为JavaScript会改变DOM和样式，以改变渲染树。

渲染树在渲染的过程中，会给元素进行布局（layout）。

DOM Tree 与 Render Tree 的联系

一般情况下，DOM树和渲染树是一一对应的。

但是，因为渲染树里都是可见的元素，当DOM树里包含了不可见的元素的时候（如display:none），就不再渲染树一一对应了。

而且设置了定位的元素，会脱离常规的渲染过程，在Render Tree里只是一个占位符，而去其他地方渲染。

重排与重绘

重排与重绘又叫 reflow 和 repaint 。

  • 重排（一般是涉及到几何属性的改变）
    • 页面第一次渲染
    • 浏览器窗口尺寸该Bain
    • 元素位置和尺寸
    • 新增和删除可见元素
    • 内容改变
  • 重绘（有重排一定有重绘，但以下情况除外）
    • 颜色属性的改变，如color、background-color
